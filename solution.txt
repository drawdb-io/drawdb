Q1. Which files in the codebase define the database datatypes? How many data types are there?
The database datatypes are defined in the file src/data/datatypes.js. There are a total of 176 datatype definitions in this file, covering various database engines including MySQL, PostgreSQL, SQLite, MSSQL, Oracle, and MariaDB.

The project does not currently use any testing framework (no testing dependencies are listed in package.json, and no test files were found in the codebase). Consequently, there are 0 tests in the project.

Q3. What parts of the codebase are responsible for rendering the tables? Which files manage the layout of tables?
The rendering of tables is primarily handled by the src/components/EditorCanvas/Table.jsx component. The layout of the tables (their positioning and arrangement) is managed by the src/utils/arrangeTables.js utility file and the src/components/EditorCanvas/Canvas.jsx component, which coordinates the overall rendering and interaction on the diagram canvas. The src/context/LayoutContext.jsx file is responsible for managing the state of the layout.
